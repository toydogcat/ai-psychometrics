# 課程三：第 4 週 - 邏輯回歸：機率建模與分類基礎 (Logistic Regression)

雖然名稱包含「回歸」，但邏輯回歸實質上是用於解決「分類」問題的最經典模型。本週我們將學習如何將線性輸出轉化為機率，並理解二分類背後的數學邏輯。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 從回歸到分類：Sigmoid 函數
- **線性模型的局限**：線性回歸輸出範圍為 $(-\infty, \infty)$，不適合表示機率。
- **Sigmoid 函數**：$\sigma(z) = \frac{1}{1 + e^{-z}}$，將任何數值映射到 $(0, 1)$ 區間。
- **物理意義**：輸出值可視為樣本屬於正類別的「機率」。

### 2. 邏輯回歸的模型表達式
- **公式**：$P(y=1|x) = \sigma(w^T x + b)$。
- **決策邊界 (Decision Boundary)**：當 $P \ge 0.5$（即 $w^T x + b \ge 0$）時判定為正類。這意味著邏輯回歸是一個線性分類器。

### 3. 損失函數：對數損失 (Log Loss)
- **為什麼不使用 MSE？**：邏輯回歸配合 MSE 會導致損失函數是非凸的，難以優化。
- **對數損失**：基於最大似然估計 (MLE)，當預測機率越偏離真實標籤時，懲罰越重。
- **公式**：$- [y \log(\hat{y}) + (1-y) \log(1-\hat{y})]$。

### 4. 正則化 (Regularization) 簡介
- **過擬合風險**：當特徵過多時，模型會過度擬合訓練數據。
- **L1 (Lasso)**：傾向於產生稀疏權重（部分權重變為 0），具備特徵選擇功能。
- **L2 (Ridge)**：傾向於讓權重趨近於 0 但不為 0，防止權重過大。

### 5. 多分類擴展
- **OvR (One-vs-Rest)**：將 $N$ 個類別拆分為 $N$ 個二分類問題。
- **Softmax**：邏輯回歸在多分類上的推廣，確保所有類別機率和為 1。

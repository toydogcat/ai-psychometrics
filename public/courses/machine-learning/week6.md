## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 集成學習 (Ensemble Learning) 核心思想
- **偏差 (Bias) 與 方差 (Variance) 分解**：模型誤差 = 偏差 + 方差 + 雜訊。
- **目標**：透過組合多個弱學習器 (Weak Learners) 來降低偏差或方差。

### 2. Bagging 原理：Bootstrap Aggregating
- **自助抽樣 (Bootstrapping)**：有放回地隨機抽取樣本。約有 36.8% 的資料不會被抽到（稱為 Out-of-Bag, OOB）。
- **聚合 (Aggregating)**：分類問題採投票制 (Voting)，回歸問題採平均制 (Averaging)。
- **核心優勢**：並行執行，有效降低方差。

### 3. 隨機森林 (Random Forest)
- **改進點**：在 Bagging 的基礎上，增加「特徵隨機化」。
- **邏輯**：在每個節點分裂時，僅從隨機選取的 $k$ 個特徵中尋找最佳切分點。
- **解耦**：進一步降低樹與樹之間的相關性，使集成效果更佳。

### 4. 袋外錯誤率 (OOB Error)
- **定義**：利用未參與訓練的 OOB 樣本進行自我驗證。
- **價值**：不需要額外的交叉驗證即可估計泛化誤差。

### 5. 極隨機樹 (Extra Trees)
- **特點**：不僅特徵隨機，切分閾值也隨機。方差更低，但偏差可能略增。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：手寫數字辨識對比
1.  **資料準備**：
    - 使用 `load_digits` 資料集（8x8 圖像展平為 64 維向量）。
2.  **單模型與集成模型對比**：
    - 訓練一個 `DecisionTreeClassifier`。
    - 訓練一個 `RandomForestClassifier` (n_estimators=100)。
    - 比較兩者的交叉驗證分數。
3.  **參數調優探討**：
    - 觀察 `n_estimators`（樹的數量）增加時，效能的變化趨勢。
    - 測試 `max_features` 對結果的影響。
4.  **OOB 驗證**：
    - 開啟 `oob_score=True`，獲取模型自帶的驗證分數。

# 課程三：第 14 週 - 特徵工程：資料清洗與特徵轉換 (Feature Engineering)

「資料與特徵決定了機器學習的上限，而模型與算法只是逼近這個上限。」本週我們將學習如何將原始數據轉化為模型易於理解的形式，涵蓋缺失值處理、特徵縮放、類別編碼等核心技術。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 資料清洗 (Data Cleaning)
- **缺失值處理**：
    - 刪除 (Drop)：樣本量大且缺失少時。
    - 均值/中位數填充：數值型特徵。
    - 眾數填充：類別型特徵。
- **異常值處理**：利用箱線圖 (IQR) 或 Z-score 識別並處理極端值。

### 2. 特徵縮放 (Feature Scaling)
- **為什麼需要縮放？**：梯度下降收斂速度與距離法模型 (KNN/SVM) 的公平性。
- **標準化 (Standardization)**：$z = (x - \mu) / \sigma$，均值為 0，方差為 1。
- **歸一化 (Normalization)**：$x' = (x - min) / (max - min)$，映射到 [0, 1]。

### 3. 類別特徵處理 (Categorical Encoding)
- **序號編碼 (Label Encoding)**：適合具備順序關係的特徵（如：大、中、小）。
- **獨熱編碼 (One-Hot Encoding)**：適合無順序關係的特徵（如：顏色、性別），防止模型誤判數值大小意義。

### 4. 數值特徵轉換
- **對數轉換 (Log Transform)**：處理偏態分佈 (Skewed Distribution)，使資料更接近常態分佈。
- **特徵離散化 (Discretization)**：將連續變數轉換為區間（分桶）。

### 5. 特徵衍生 (Feature Creation)
- **多項式特徵**：增加特徵間的交互作用。
- **領域知識**：從日期中提取「星期幾」、「是否為假日」。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：打造乾淨的資料流水線
1.  **資料準備**：
    - 使用 `load_iris` 或自行產生一個帶有缺失值與類別特徵的 DataFrame。
2.  **Imputation 實作**：
    - 使用 `sklearn.impute.SimpleImputer` 填充缺失值。
3.  **Encoding 與 Scaling**：
    - 使用 `OneHotEncoder` 處理分類變數。
    - 使用 `StandardScaler` 進行特徵縮放。
4.  **對比分析**：
    - 訓練一個模型，對比「特徵縮放前」與「特徵縮放後」的訓練速度與準確率。
5.  **視覺化**：
    - 繪製對數轉換前後的資料分佈直方圖。

# 課程三：第 13 週 - 異常檢測：孤立森林與局部異常因子 (Anomaly Detection)

異常檢測旨在從海量數據中找出那些「與眾不同」的少數點。本週我們將學習如何不依賴標籤，利用數據的密度與結構發現異常，並應用於金融詐騙、網路攻擊等實戰場景。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 什麼是異常檢測 (Anomaly Detection)？
- **定義**：識別不符合預期模式或資料集中其他項目的樣本。
- **類型**：點異常、上下文異常、集體異常。
- **挑戰**：類別極度不平衡（異常極少），標籤往往缺失。

### 2. 孤立森林 (Isolation Forest)
- **核心思想**：異常點通常「少且遠」，因此更容易被孤立。
- **算法流程**：
    - 隨機選擇特徵，隨機選擇切分點。
    - 異常樣本在決策樹中通常位於較短的路徑。
- **優勢**：適合高維資料，計算效率極高。

### 3. 局部異常因子 (Local Outlier Factor, LOF)
- **核心思想**：比較一個點與其鄰居的「局部密度」。
- **邏輯**：如果一個點的密度遠低於其鄰居的密度，則該點極有可能是異常。
- **優勢**：擅長處理資料集中密度分佈不均的情況。

### 4. 一類 SVM (One-Class SVM) 簡介
- **原理**：在特徵空間中尋找一個包含大多數正常點的超球體或邊界，邊界外的點視為異常。

### 5. 評估異常檢測模型
- **矛盾指標**：準確率 (Accuracy) 毫無意義。
- **重點指標**：精準率 (Precision)、召回率 (Recall) 與 $F1$-Score。
- **污染率 (Contamination)**：預估資料集中異常值的比例。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：信用卡詐騙偵測模擬
1.  **資料模擬**：
    - 使用 `make_blobs` 產生大量正常點。
    - 手動加入少量遠離中心的異常點。
2.  **孤立森林實作**：
    - 使用 `IsolationForest` 進行預測。
    - 視覺化「正常點」與「被標記異常點」的分佈圖。
3.  **密度法實驗**：
    - 使用 `LocalOutlierFactor` 處理多密度簇的數據，觀察其與孤立森林的結果差異。
4.  **調參挑戰**：
    - 修改 `contamination` 參數，觀察對 F1-Score 的影響。
    - 繪製異常分數 (Anomaly Score) 的直方圖。

## 三 island 知識點回顧 (KP)

- **[KP 3.13.1] 異常的本質特徵**：理解異常點具備「疏離性」與「局部低密度性」。
- **[KP 3.13.2] 孤立路徑長度**：明白為什麼路徑越短（切分次數越少）越可能是異常。
- **[KP 3.13.3] 局部密度比值**：掌握 LOF 如何透過比較密度比來發現異常。
- **[KP 3.13.4] 非平衡評估思維**：牢記在異常檢測中，捕捉到異常 (Recall) 通常比預測對正常更重要。
- **[KP 3.13.5] 污染率參數調優**：學會根據業務背景設定 `contamination` 的初始值。

# 課程三：第 11 週 - 主成分分析 (PCA)：線性降維的藝術 (PCA)

高維資料不僅難以視覺化，還會帶來計算災難（維度災難）。本週我們將學習主成分分析 (PCA)，探討如何透過線性映射將資料投影至低維空間，並同時保留最多的資訊。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 為什麼需要降維？
- **視覺化**：人類只能理解 2D 或 3D 空間。
- **去噪**：移除不重要的微小變異。
- **壓縮**：節省存儲與計算資源。
- **特徵提取**：組合多個相關特徵為獨立的主成分。

### 2. PCA 的核心思想
- **最大化方差**：投影後的方向上，資料分佈越散，保留的資訊越多。
- **最小化投影誤差**：原始點到投影平面的距離和最小。

### 3. PCA 運算流程
- **中心化**：將特徵均值移至 0（平移座標系）。
- **計算協方差矩陣**：衡量特徵間的相關性。
- **特徵分解**：求出協方差矩陣的特徵向量（主成分方向）與特徵值（方差大小）。
- **投影**：選擇前 $k$ 個主成分進行線性組合。

### 4. 主成分的特性
- **正交性**：各主成分之間互不相關（點積為 0）。
- **重要性遞減**：第一主成分解釋最多的方差，依此類推。

### 5. 如何選擇主成分數量？
- **解釋方差比 (Explained Variance Ratio)**：單個成分佔總變異的比例。
- **累計解釋方差**：通常保留能解釋 90%-95% 以上變異的成分數量。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：高維資料視覺化
1.  **資料探索**：
    - 使用 `load_iris` (4維) 或 `load_breast_cancer` (30維) 資料集。
2.  **PCA 實作**：
    - 使用 `sklearn.decomposition.PCA` 將資料降至 2 維。
3.  **視覺化分析**：
    - 繪製 2D 散點圖，觀察不同類別在主成分空間中的分離程度。
4.  **選擇最佳組件數**：
    - 繪製「碎石圖 (Scree Plot)」，觀察累計解釋方差隨維度增加的變化。
5.  **對比實驗**：
    - 比較「原始資料訓練模型」與「PCA 降維後訓練模型」的速度與準確率。

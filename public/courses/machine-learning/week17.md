# 課程三：第 17 週 - 超參數調優與自動化管線 (Hyperparameter Tuning & Pipelines)

「手動調參」既低效又容易產生資料滲漏。本週我們將學習如何利用網格搜索與隨機搜索自動尋找模型的最優超參數。同時，我們將學習 Scikit-Learn 的核心武器——Pipeline，將預處理與建模封裝成一個不可分割的整體。

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 參數 (Parameters) vs. 超參數 (Hyperparameters)
- **參數**：模型內部學到的權重（如 $w, b$）。
- **超參數**：訓練前設定的控制項（如樹的深度、學習率、C 值）。

### 2. 網格搜索 (Grid Search)
- **原理**：設定候選值清單，窮舉所有可能的參數組合。
- **優缺點**：保證在設定範圍內找到最優，但面對參數多時會發生「維度災難」。

### 3. 隨機搜索 (Randomized Search)
- **原理**：在指定的參數分佈中隨機取樣 $N$ 組。
- **優勢**：運算成本可控，且在重要參數分佈不均時，往往比網格搜索更有效率。

### 4. Scikit-Learn Pipeline
- **痛點**：如果在 Split 前做標準化，測試集的資訊會透過均值滲漏給訓練集。
- **解決方案**：將 `Scaler` 與 `Model` 串聯。Pipeline 的 `fit` 會自動在訓練集上計算均值，`predict` 時自動套用。
- **代碼潔癖**：大幅簡化工作流，防止出錯。

### 5. 模型持久化 (Serialization)
- **工具**：`joblib` 與 `pickle`。
- **應用**：將訓練好的模型保存為檔案，以便在生產環境中載入預測。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：自動化建模大賽
1.  **構建 Pipeline**：
    - 建立一個包含 `SimpleImputer` (填充缺失值)、`StandardScaler` 與 `RandomForestClassifier` 的 Pipeline。
2.  **執行 GridSearchCV**：
    - 定義參數網格（如 `rf__n_estimators`, `rf__max_depth`）。
    - 在 Pipeline 上直接執行搜索。
3.  **比較 RandomizedSearchCV**：
    - 對比兩者的執行時間與獲得的最佳分數。
4.  **保存與載入模型**：
    - 將最優模型保存為 `.pkl` 檔。
    - 模擬重啟 Python，載入檔案並對新數據進行 `predict`。

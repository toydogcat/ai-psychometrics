# 課程三：第 5 週 - 決策樹：資訊理論與分而治之 (Decision Trees)

決策樹是一種直觀且具備高度可解釋性的模型。本週我們將學習如何利用「熵」與「基尼不純度」來構建決策邏輯，並探討如何防止樹模型過度生長。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 決策樹的結構
- **組成要素**：根節點 (Root)、內部節點 (Internal Nodes)、分支 (Branches) 與葉節點 (Leaf Nodes)。
- **預測邏輯**：透過一系列的 `if-else` 判斷，將特徵空間劃分為多個矩形區域。

### 2. 核心指標：不純度 (Impurity)
- **資訊熵 (Entropy)**：衡量資訊混亂程度。$H(S) = -\sum p_i \log_2 p_i$。
- **資訊增益 (Information Gain)**：切分前後熵的差值。ID3 算法的核心。
- **基尼不純度 (Gini Impurity)**：從集合中隨機抽取兩個樣本，其類別不一致的機率。CART 算法的核心。

### 3. CART 算法與切分策略
- **二元切分**：CART 樹始終是二元樹。
- **連續特徵處理**：透過尋找最佳閾值將連續變數離散化。

### 4. 過擬合與剪枝 (Pruning)
- **問題**：樹長得太深會完全記住訓練集（高方差）。
- **預剪枝 (Pre-pruning)**：限制 `max_depth`, `min_samples_leaf` 等。
- **後剪枝 (Post-pruning)**：先讓樹長完，再移除貢獻度低的節點。

### 5. 特徵重要性 (Feature Importance)
- **原理**：根據特徵在減少不純度上的總體貢獻來排名。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：視覺化決策邊界
1.  **資料準備**：
    - 使用 `load_iris` 或 `make_moons` 產生非線性資料。
2.  **模型訓練與參數對比**：
    - 訓練一個無限制的樹 vs. 限制 `max_depth=3` 的樹。
    - 觀察兩者在測試集上的準確率差異。
3.  **樹的視覺化**：
    - 使用 `export_graphviz` 或 `plot_tree` 繪製決策路徑。
    - 解讀根節點選擇了哪個特徵，以及為什麼。
4.  **特徵重要性排名**：
    - 繪製橫向柱狀圖展示 `feature_importances_`。

# 課程一：第 15 週 - 正規表示式 (Regular Expressions)

正規表示式是處理字串強大且通用的工具。本週我們將學習正規表示式的語法規範，以及如何在 Python 中使用 `re` 模組進行精準的文字搜尋、提取、替換與驗證。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 正規表示式基礎語法
- **字元類別**：`.` (任意字元), `\d` (數字), `\D` (非數字), `\w` (英數字下劃線), `\s` (空白字元)。
- **錨點 (Anchors)**：`^` (行首), `$` (行尾), `\b` (單字邊界)。
- **集合與排除**：`[abc]` (匹配 a, b 或 c), `[^abc]` (排除 a, b, c), `[a-z]` (範圍匹配)。

### 2. 量詞 (Quantifiers) 與貪婪模式
- **量詞語法**：`*` (0+), `+` (1+), `?` (0 or 1), `{n,m}` (次數範圍)。
- **貪婪 (Greedy) vs. 非貪婪 (Non-greedy)**：預設匹配最長字串，加上 `?` 變為匹配最短（如 `.*?`）。

### 3. Python `re` 模組核心函式
- **`re.match()`**：從字串開頭匹配。
- **`re.search()`**：掃描整個字串並回傳第一個匹配。
- **`re.findall()`**：回傳所有匹配項目的串列表。
- **`re.finditer()`**：回傳一個包含匹配物件的疊代器。
- **`re.sub()`**：字串替換。

### 4. 捕獲群組 (Capture Groups)
- **括號應用**：使用 `()` 進行分組。
- **提取資訊**：透過 `group()` 或 `groups()` 獲取分組內容。
- **命名群組**：`(?P<name>...)` 語法，提高程式碼可讀性。
- **非捕獲群組**：`(?:...)` 的應用場景。

### 5. 編譯正規表示式與旗標 (Flags)
- **`re.compile()`**：預先編譯以提升效率（適合在迴圈中使用）。
- **常用旗標**：`re.IGNORECASE` (忽略大小寫), `re.MULTILINE` (多行模式), `re.DOTALL` (讓 `.` 匹配換行符)。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：開發「智慧文字處理器」
1.  **實作「電子郵件驗證器」**：
    - 撰寫一個正規表示式，檢查使用者輸入是否為合法的 Email 格式。
    - 支援 `user@domain.com` 格式。
2.  **實作「Log 資訊提取工具」**：
    - 給定一段伺服器日誌（包含日期、時間、IP 地址與錯誤訊息）。
    - 使用 `re.findall` 提取出所有的 IP 地址。
    - 使用命名群組分別提取「日期」與「時間」。
3.  **實作「敏感詞遮罩器」**：
    - 指定一組禁用語清單。
    - 使用 `re.sub` 將字串中的禁用語替換為 `***`。
4.  **網址 (URL) 解析器**：
    - 使用 `re.search` 搭配捕獲群組，從 URL 中提取「協定」(http/https)、「網域名稱」與「路徑」。

## 三、 本週知識點回顧 (KP)

- **[KP 15.1] 基礎語法元字元**：了解點號、星號、加號等符號在 Regex 中的特殊含義。
- **[KP 15.2] 搜尋 vs. 匹配**：`re.search` 查找全字串，`re.match` 僅從開頭檢查，使用時需分清。
- **[KP 15.3] 捕獲群組**：分組不僅是為了邏輯封裝，更是為了精確提取子字串。
- **[KP 15.4] 貪婪與非貪婪**：處理 HTML 標籤或成對符號時，非貪婪模式 `.*?` 是避免匹配過度的關鍵。
- **[KP 15.5] 編譯優化**：對於頻繁使用的 Regex，使用 `re.compile` 是效能優化的基本常識。

### 1. 單選題 (Single Choice) - 10 題

2. **Q2**: 下列哪一個量詞代表「出現一次或多次」？
   - (A) `*` (B) `?` (C) `+` (D) `{1}`
3. **Q3**: `re.match('abc', '123abc')` 的回傳值是？
   - (A) 匹配物件 (B) `'abc'` (C) `None` (D) 拋出異常
4. **Q4**: 若要讓匹配不區分大小寫，應在 `re.search()` 中加入哪個旗標？
   - (A) `re.I` (B) `re.M` (C) `re.S` (D) `re.X`
5. **Q5**: 如何在 Regex 中表示字串的結尾？
   - (A) `^` (B) `&` (C) `$` (D) `\z`
6. **Q6**: 執行 `re.sub(r'\d', '#', 'Age: 25')` 的結果是？
   - (A) `'Age: ##'` (B) `'Age: #'` (C) `'#ge: 25'` (D) `'Age: 25'`
7. **Q7**: Regex `\w` 等價於下列哪一個集合？

8. **Q8**: 哪一個函式會回傳所有匹配結果的疊代器 (Iterator)？
   - (A) `re.findall()` (B) `re.search()` (C) `re.finditer()` (D) `re.match()`
9. **Q9**: 想要匹配字面上的問號 `?`，應該寫成？
   - (A) `?` (B) `\?` (C) `[?]` (D) B 或 C 均可
10. **Q10**: 關於 `re.compile()`，下列敘述何者正確？
    - (A) 它會把字串轉為 Python 程式碼 (B) 它可以加速重複使用的 Regex (C) 它只能用在 `match` 中 (D) 它會自動加上旗標

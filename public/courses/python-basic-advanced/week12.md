# 課程一：第 12 週 - 進階函式與裝飾器 (Decorators)

本週我們將深入探討 Python 中「函式是一等公民」的極致應用。從閉包的記憶機制到裝飾器的切面編程 (AOP)，學習如何優雅地擴充現有程式碼的功能而不修改其核心邏輯。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 閉包 (Closures) 與 `nonlocal` 關鍵字
- **閉包的定義**：一個能記住其定義時環境（作用域）的函式。
- **自由變數 (Free Variables)**：存在於外部函式但被內部函式引用的變數。
- **`nonlocal`**：當內部函式需要修改外部（非全域）變數時的聲明。

### 2. 裝飾器 (Decorators) 的基本原理
- **語法糖 `@`**：裝飾器本質上是一個接收函式並回傳新函式的高階函式。
- **基本結構**：`outer` -> `inner(*args, **kwargs)` -> `return inner`。
- **執行時機**：裝飾器是在函式定義時執行的，而非呼叫時。

### 3. 帶參數的裝飾器 (Nested Decorators)
- **需求**：如果我們希望裝飾器本身能接收參數（例如：`@retry(times=3)`）。
- **三層結構**：最外層接收參數，中間層接收函式，最內層處理執行邏輯。

### 4. `functools.wraps` 的重要性
- **副作用**：被裝飾後的函式，其 `__name__` 與 `__doc__` 會變成裝飾器的內部函式名。
- **解決方案**：使用 `@functools.wraps(func)` 裝飾內部函式，以保留原始函式的元資料 (Metadata)。

### 5. 裝飾類別中的方法
- **實體方法裝飾**：注意 `self` 參數的傳遞。
- **多重裝飾器**：疊加順序（由內而外裝飾，由上而下執行）。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：開發「系統監控與權限裝飾器組」
1.  **實作 `@timer` 裝飾器**：
    - 紀錄函式的開始與結束時間，印出執行耗時。
    - 使用 `functools.wraps`。
2.  **實作 `@debug` 裝飾器**：
    - 印出函式的名稱及其收到的所有參數值。
3.  **實作 `@rate_limit(seconds)` 裝飾器**：
    - 限制函式在指定秒數內只能執行一次，否則印出 "Too fast!"。
    - 提示：需結合「閉包」來紀錄上次執行的時間戳。
4.  **綜合應用**：
    - 同時在一個計算密集型函式上套用多個裝飾器，觀察輸出順序。

- **[KP 12.1] 閉包機制**：函式與環境的綁定，使得變數在外部函式結束後仍能存活。
- **[KP 12.2] 裝飾器本質**：`func = decorator(func)` 的語法簡寫，實現非侵入式功能擴充。
- **[KP 12.3] 參數解包應用**：利用 `*args` 與 `**kwargs` 確保裝飾器能相容任何參數配置。
- **[KP 12.4] 元資料保護**：習慣性使用 `wraps` 以維持偵錯時的可讀性。
- **[KP 12.5] 帶參裝飾器邏輯**：多一層閉包封裝，用於動態調整裝飾器行為。

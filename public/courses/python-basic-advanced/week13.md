# 課程一：第 13 週 - 疊代器與產生器 (Iterators & Generators)

本週我們將探索 Python 處理大數據集的利器：疊代器與產生器。了解如何透過「延遲求值」(Lazy Evaluation) 機制，在有限的記憶體空間內處理無限或超大型的資料流。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 疊代協定 (Iteration Protocol)
- **可疊代物件 (Iterable)**：定義了 `__iter__` 方法，能回傳一個疊代器（如 list, str, dict）。
- **疊代器 (Iterator)**：定義了 `__next__` 方法，能回傳序列中的下一個值。
- **StopIteration**：疊代結束時拋出的例外訊號。

### 2. 實作自定義疊代器類別
- 建立一個類別，同時實作 `__iter__` 與 `__next__`。
- 狀態保持：如何在類別實例中紀錄目前的進度。
- 範例：實作一個 `ReverseRange` 倒序疊代器。

### 3. 產生器 (Generators) 與 `yield` 關鍵字
- **產生器函式**：包含 `yield` 的函式。呼叫時不執行，而是回傳一個產生器物件。
- **`yield` 的行為**：暫停執行並回傳值，紀錄目前的暫存器與局部變數狀態，下次呼叫從原位恢復。
- **與 `return` 的區別**：`return` 徹底結束函式；`yield` 只是暫停。

### 4. 產生器表達式 (Generator Expressions)
- **語法**：使用圓括號 `(x for x in data)`。
- **與串列生成式對比**：產生器表達式不立即在記憶體中建立整個列表。
- **組合應用**：在 `sum()`, `max()`, `min()` 等聚合函式中直接使用。

### 5. 記憶體效率與延遲求值 (Lazy Evaluation)
- **空間複雜度分析**：`range(1000000)` vs `list(range(1000000))`。
- **處理超大型檔案**：為什麼應該使用 `for line in file` 而非 `file.readlines()`。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：開發「高效能大數據處理工具」
1.  **實作「費波那契 (Fibonacci)」產生器**：
    - 撰寫一個函式，能產生無限的費波那契數列。
    - 使用 `next()` 獲取前 10 個數字。
2.  **實作「檔案內容過濾器」**：
    - 撰寫一個產生器，逐行讀取一個大型 Log 檔案。
    - 僅 `yield` 包含 "ERROR" 字樣的行數。
3.  **串聯產生器 (Pipelining)**：
    - 建立一個產生器流水線：`讀取檔案` -> `過濾關鍵字` -> `轉換為大寫` -> `統計行數`。
    - 觀察整個過程中記憶體的佔用情況（理論分析）。
4.  **對比測試**：
    - 計算 1 到 10,000,000 的平方和。
    - 分別使用「串列生成式」與「產生器表達式」，觀察啟動速度與記憶體峰值（可使用 `sys.getsizeof`）。

## 三、 本週知識點回顧 (KP)

- **[KP 13.1] 疊代器協定**：凡是實作了 `__next__` 且回傳自身的 `__iter__` 物件皆為疊代器。
- **[KP 13.2] 狀態機特性**：產生器本質上是一個自動管理的狀態機。
- **[KP 13.3] yield 暫停機制**：函式不再是一次性執行，而是可以多次進入、多次離開。
- **[KP 13.4] 產生器表達式**：針對一次性消費數據的極致優化手段。
- **[KP 13.5] 延遲計算 (Lazy)**：將計算延後到真正需要數據的那一刻，大幅節省記憶體。

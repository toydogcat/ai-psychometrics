# 課程五：第 10 週 - 提示工程基礎：溝通的藝術 (Prompt Engineering)

給予正確的指令，AI 就是天才；指令錯誤，AI 就是庸才。本週我們將學習提示工程 (Prompt Engineering) 的核心要素，掌握如何透過角色設定、上下文填充與格式控制，精準引導大語言模型 (LLM) 產出高品質結果。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 提示工程 (Prompt Engineering) 的定義
- **本質**：將人類意圖轉化為模型最易理解的文字指令。
- **目標**：提高輸出準確度、穩定格式、減少幻覺。

### 2. Prompt 的六大核心要素 (CO-STAR 框架)
- **C (Context)**：背景資訊。提供任務發生的環境。
- **O (Objective)**：目標。明確要求模型做什麼。
- **S (Style)**：風格。指定寫作風格（如：幽默、專業）。
- **T (Tone)**：語氣。設定情感基調（如：親切、嚴肅）。
- **A (Audience)**：讀者。誰會看到這份結果？
- **R (Response)**：格式。指定輸出 JSON, Table 或 Markdown。

### 3. 提示技巧：Zero-shot 與 Few-shot
- **零樣本 (Zero-shot)**：直接下指令（適合簡單、常識性任務）。
- **少樣本 (Few-shot)**：給予 1-3 個範例（適合複雜邏輯或特定格式要求）。

### 4. 角色設定 (Persona) 的威力
- **原理**：透過設定「你是資深架構師」或「你是五歲小孩」，模型會自動調用關聯的語義空間，調整專業深度。

### 5. 指令分隔符與負面約束
- **分隔符**：使用 `###` 或 `---` 區分指令與原始資料，防止模型混淆。
- **約束**：明確告訴模型「不要做什麼」（如：不要使用專業術語）。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：Prompt 優化大挑戰
1.  **從模糊到精準**：
    - **原始指令**：寫一篇關於 AI 的文章。
    - **任務**：將其修改為包含角色、背景、目標讀者與字數限制的高品質 Prompt。
2.  **格式化控制賽**：
    - 撰寫一個 Prompt，輸入 5 位員工的打卡時間，要求模型以 JSON 格式輸出其工時統計。
    - 驗證模型輸出的 JSON 格式是否正確。
3.  **Few-shot 實踐**：
    - 任務：將白話文翻譯成特定的「江湖語氣」。
    - 步驟：提供 2 個江湖範例，然後輸入新句子觀察生成效果。
4.  **反向 Prompt 練習**：
    - 展示一段高品質的 AI 生成內容，讓學生嘗試推導出「背後的原始 Prompt 是什麼」。

## 三 island 知識點回顧 (KP)

- **[KP 5.10.1] Prompt 結構化思維**：掌握背景、任務、輸出格式三位一體的撰寫習慣。
- **[KP 5.10.2] 角色設定的心理學效應**：理解 Persona 如何限縮模型的生成範圍。
- **[KP 5.10.3] 少樣本學習的應用時機**：學會透過範例解決模型無法理解的抽象格式要求。
- **[KP 5.10.4] 輸出格式鎖定技術**：明白如何強制模型產出機器可讀（JSON/CSV）的格式。
- **[KP 5.10.5] 提示詞迭代策略**：掌握「觀察 -> 修改 -> 測試」的螺旋式優化流程。

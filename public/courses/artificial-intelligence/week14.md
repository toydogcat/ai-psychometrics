# 課程五：第 14 週 - 低代碼/無代碼 AI 解決方案開發 (No-code AI)

不寫程式也能建構 AI 應用？本週我們將認識「無代碼 (No-code)」與「低代碼 (Low-code)」開發工具。你將學會如何透過拖拉拽的方式，將 AI 模型、資料庫與自動化流程串接起來，快速產出商業原型 (MVP)。

---

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 為什麼需要無代碼 AI 開發？
- **速度**：從想法到上線只需幾小時而非數週。
- **門檻**：讓業務人員 (PM/Planner) 也能直接參與技術實現。
- **維護**：視覺化介面降低了理解與交接的難度。

### 2. OpenAI GPTs 與 個人化助理
- **自定義指令 (System Prompt)**：賦予助理特定行為。
- **知識庫掛載**：讓助理只回答你上傳的 PDF 內容（簡易 RAG）。
- **能力組合**：聯網、繪圖、代碼執行器的開關。

### 3. 自動化工作流串接：Make.com 與 Zapier
- **觸發器 (Trigger)**：事件的起點（如：收到一封 Gmail）。
- **行動 (Action)**：AI 的介入（如：讓 GPT 總結郵件並存入 Google Sheet）。
- **邏輯分叉**：如果情感是負面的，則通知主管；如果是正面的，則自動回信。

### 4. 低代碼平台概覽
- **Microsoft Copilot Studio**：企業級對話機器人開發。
- **Coze (扣子)**：強大的外掛 (Plugins) 生態與多代理 (Multi-agent) 協作。

### 5. MVP (最小可行性產品) 思維
- **規劃**：不要一開始就想做完美的 App，先解決一個核心痛點。
- **成本**：學會估算第三方工具的月租與 API 消耗。

## 二、 動手實作 (Lab) - 50 分鐘

### 任務：打造你的自動化 AI 助理
1.  **GPTs 實戰**：
    - 在 OpenAI 平台上建立一個「ipas 認證助手」。
    - 上傳本課程的大綱文件作為知識庫。
    - 測試模型是否能根據文件內容為你出一道考題。
2.  **自動化流程設計 (以 Make 或 Zapier 模擬)**：
    - 繪製一個流程圖：當客戶在 Google Form 提交問題 -> AI 自動分類 -> 存入 Notion 資料庫。
3.  **多代理協作體驗 (Coze)**：
    - 在 Coze 平台建立一個機器人。
    - 學習如何添加「Google 搜尋」與「DALL-E 繪圖」插件。
    - 測試機器人是否能幫你「查新聞並配圖」。
4.  **成本試算表**：
    - 假設每月有 1000 封郵件需要 AI 處理。
    - 計算選用 OpenAI API vs. 買專用軟體的成本差異。

## 三 island 知識點回顧 (KP)

- **[KP 5.14.1] 無代碼開發的價值鏈**：理解其在快速驗證商業假說中的關鍵角色。
- **[KP 5.14.2] 指令、知識與能力的配置**：掌握構建自定義助理的三大支柱。
- **[KP 5.14.3] API 驅動的自動化邏輯**：明白 Webhook、觸發器與行動的串接原理。
- **[KP 5.14.4] 插件 (Plugins) 的延伸功能**：掌握如何透過外部工具補足 LLM 的能力。
- **[KP 5.14.5] 解決方案的技術債**：識別無代碼工具在擴展性與數據安全性上的潛在局限。

# 課程四：第 13 週 - 注意力機制 (Attention)：AI 的集中力 (Attention Mechanism)

人類看書時不會每顆字都給予同樣的關注，AI 也應如此。本週我們將探討深度學習歷史上最重要的突破之一：注意力機制。我們將拆解其內部的 Q, K, V 邏輯，看它如何打破 RNN 的性能天花板。

## 一、 單元講解 (Lecture) - 100 分鐘

### 1. 注意力機制的直覺
- **資源分配**：不要試圖記住整句話，要關注與當前任務最相關的部分。
- **解決資訊瓶頸**：打破 Seq2Seq 中 Context Vector 長度固定的限制。

### 2. 核心公式：Query, Key, Value
- **Query (查詢)**：你想找什麼？
- **Key (鍵)**：每個候選資訊的特徵是什麼？
- **Value (值)**：候選資訊的實際內容。
- **打分函數**：點積 (Dot-product) 與 縮放點積 (Scaled Dot-product)。

### 3. 計算三部曲
- **第一步：相似度計算**：Q 與各個 K 計算匹配得分。
- **第二步：歸一化**：透過 Softmax 將得分轉化為權重（和為 1）。
- **第三步：加權求和**：根據權重將 V 組合起來。

### 4. 自注意力 (Self-Attention)
- **定義**：輸入序列中的每個元素都去查詢序列中的其他元素。
- **意義**：捕捉句子內部的長程依賴關係（例如代詞「它」指代的是前面的哪個詞）。

### 5. 多頭注意力 (Multi-head Attention) 初探
- **並行觀察**：像多個攝像頭從不同角度看同一場景（語法角度、語義角度）。

### 任務：手動實作縮放點積注意力
1.  **矩陣運算實踐**：
    - 在 PyTorch 中手動編寫 `QK^T / sqrt(dk)` 的矩陣乘法。
2.  **注意力圖可視化**：
    - 繪製 Heatmap，觀察在機器翻譯時，目標詞如何自動對齊到源詞。
3.  **對比實驗**：
    - 實現一個帶 Attention 的 Seq2Seq 模型。
    - 觀察其在長句子（50 字以上）翻譯時，相比傳統 RNN 的明顯優勢。
4.  **遮罩 (Masking) 測試**：
    - 學習如何手動將 Padding 部分的得分設為 $-\infty$，使其 Softmax 後變為 0。
